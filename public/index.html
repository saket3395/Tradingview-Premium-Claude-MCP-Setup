      How it works
      
            
        
            Stage 1 — broad scan (automatic). Every cycle the server queries TradingView's public
        scanner for the entire NSE universe (~2,900 primary stocks) and scores each one. Data is
        real but typically ~15 min delayed (real-time NSE needs your logged-in session, which the
        browser blocks cross-origin). No chart is touched — the scan runs server-side.
            
        
            Conviction score (0–100) blends native Market-Profile concepts, all from real fields:
        Opening Location (gap + open-drive), Value Acceptance/Rejection (position in the day
        range and price vs the session Pivot), IB / Range-Extension trend-day proxy (range ÷ ATR),
        Relative Volume conviction, NIFTY alignment, and R:R geometry. Only setups scoring
        ≥70 with R:R ≥1.5 and rVol ≥1.2 are shown, ranked by conviction — thresholds live in
        config/markets.json → tpo.
            
        
            Levels. Entry is a pullback into value (the Pivot) rather than chasing the extreme;
        SL sits beyond value (tighter of an ATR stop or the day extreme); Targets project range
        extension beyond the session high/low. R:R falls out of that geometry.
            
        
            Stage 2 — confirm (on demand). The Confirm button loads that symbol on your active
        TradingView chart and reads live OHLC plus any Volume/Market-Profile study present — true
        POC/VAH/VAL appear only if such a study is on the chart.
            
        
            Honest caveats: the scanner's “VWAP” field is actually the pivot
        (H+L+C)/3, so it is labeled Pivot, not VWAP. Full letter-by-letter TPO
        (POC/VAH/VAL/IB/single prints) needs intraday time-at-price and isn’t computable for the whole
        universe from day-level data — Stage 1 is profile-informed; Stage 2 gives true on-chart levels.
